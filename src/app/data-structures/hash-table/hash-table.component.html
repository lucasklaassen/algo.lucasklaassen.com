Hash Table

Toggle Description

This data structure stores key value mappings. It uses an array of singly linked lists to store the data. When storing data, each key is mapped to a hash code which is then mapped to an index of the array. Multiple keys can hash to the same index (this is called collision) so we use a singly linked list to store many key value pairs at the same index of the array. The array resizes overtime as the data grows and shrinks. When resizing the array you need to rehash all of the key value pairs since the size of the array determines what index range the hash codes can be mapped to. All actions use constant amortized time since the array needs to resize which takes linear time every once in a while.
