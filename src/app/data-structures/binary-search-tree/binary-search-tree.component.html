Binary Search Tree

Toggle Description

This data structure stores a collection of nodes in sorted order so that actions performed on it take up logarithmic time. Each node has a right and left child node, both of which can be null. For each node, the left child stores keys which are smaller than itself and the right child stores keys which are larger. Storing data this way allows you to Binary Search to insert, fetch and delete information. This means that all of it's actions take logarithmic time.
