Stack

Toggle Description

This data structure stores nodes on top of each other as if they are living on a pile. LIFO (last in first out) - You can only view and remove the last item on the pile just like real life. All actions use constant time, however since we've used a singly linked list to implement this stack, the insert and remove actions require new allocation of memory. You can also use an array to implement this data structure which may have better overall performance. You can use a simple algorthim to double the size of the array once the amount of items in the stack take up the full size of the current array. Then when items are being removed, you can remove a quarter of space once the array is storing half of it's capacity. This uses constant amortized time, which means that overall it will take constant time with a few actions taking linear time to do the reallocation of memory.
