Queue

Toggle Description

This data structure stores nodes in FIFO (first in, first out) order. You can only view and dequeue the first item you pushed to the queue (this resets to the second item you pushed to the queue after it's removed, and so on.). All actions use constant time, however since we've used a singly linked list to implement this queue, the insert and remove actions require new allocation of memory. You can also use an array to implement this data structure which may have better overall performance. You can use a simple algorthim to double the size of the array once the amount of items in the queue take up the full size of the current array. Then when items are being dequeued, you can remove a quarter of space once the array is storing half of it's capacity. This uses constant amortized time, which means that overall it will take constant time with a few actions taking linear time to do the reallocation of memory.
